# Security Policy

## Supported Release Line

| Version | Supported |
|---------|-----------|
| 1.1.x | Yes |
| Older lines | No |

## Reporting A Vulnerability

Please **do not** disclose suspected vulnerabilities in a public GitHub issue.

1. If GitHub private vulnerability reporting is enabled for this repository, use that path first.
2. If private reporting is not available, contact the repository owners through GitHub and request a private channel.
3. Share the smallest reproducible report that demonstrates impact without publishing exploit details or secrets.

Please include:

- affected version or commit
- attack surface and prerequisites
- reproduction steps
- expected impact
- any known mitigations or workarounds

Maintainers will triage reports as quickly as practical, but this repository does not currently publish a formal response SLA.
